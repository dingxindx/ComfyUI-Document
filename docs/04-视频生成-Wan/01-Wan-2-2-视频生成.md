# Wan 2.2 视频生成

> **前置**：已经掌握了 LTX 视频生成的基本概念。Wan 2.2 是阿里开源的视频生成模型系列。

## 一、模型概览

| 模型 | 参数量 | 功能 | 显存需求（fp8）|
|:----:|:------:|:----|:--------------:|
| Wan 2.2 5B T2V | 5B | 文本 → 视频 | 12-16GB |
| Wan 2.2 5B I2V | 5B | 图像 + 文本 → 视频 | 12-16GB |
| Wan 2.2 14B S2V | 14B | 声音 → 驱动视频 | 32GB+ |

## 二、前置准备

### 模型下载

使用 `hf-mirror.com` 镜像下载：

| 模型文件 | 存放路径 |
|:---------|:---------|
| `Wan2_2-T2V-5B.safetensors` | `models/checkpoints/` |
| `Wan2_2-I2V-5B.safetensors` | `models/checkpoints/` |
| `Wan2_2-VAE.safetensors` | `models/vae/` |
| `umt5_xxl.safetensors`（文本编码器） | `models/clip/` |

### 自定义节点

ComfyUI 0.3.0+ 已内置 Wan 支持，或安装第三方节点：
```bash
cd custom_nodes/
git clone https://gitclone.com/github.com/kijai/ComfyUI-Wan-Superresolution.git
```

**推荐使用 Workflow Templates**：Workflow → Browse Workflow Templates → 搜索 "Wan"

## 三、与 LTX 的关键区别

| 差异 | LTX 2.3 | Wan 2.2 |
|:-----|:---------|:---------|
| 加载方式 | CheckpointLoaderSimple（一体） | **分体加载**：UNETLoader + CLIPLoader + VAELoader |
| 文本编码器 | Gemma 3 12B | **UMT5**（非标准 CLIP）|
| 采样调度 | ModelSamplingLTXV | **ModelSamplingSD3** |

## 四、标准加载链

```
UNETLoader (Wan2_2-T2V-5B) → ModelSamplingSD3 → KSampler
CLIPLoader (umt5_xxl) → CLIP Text Encode → KSampler
VAELoader (Wan2_2-VAE) → VAE Decode
```

### ModelSamplingSD3
| 参数 | 推荐值 | 说明 |
|:-----|:------:|:-----|
| `shift` | 4.0 | 影响视频动态幅度和稳定性 |

## 五、I2V 首帧过烤修复

Wan 2.2 I2V 模式常见首帧被 AI "过烤"。修复方法：

```
Load Image → VAE Encode → LatentCut → LatentConcat → KSampler
EmptyLatentVideo ──→ LatentConcat ──┘
```

需要 `ComfyUI-KJNodes` 或类似节点包。

## 六、轻量路径：Lightning LoRA

4 步采样 + CFG≈1.0，显存降低约 30%：

| 节点 | 参数 | 值 |
|:-----|:-----|:---|
| Load LoRA | lora_name | `wan2.2-5b-lightning-lora` |
| | strength_model | 1.0 |
| KSampler | steps | 4 |
| | cfg | 1.0 |
| | sampler_name | euler |
