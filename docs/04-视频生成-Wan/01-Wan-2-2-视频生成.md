# Wan 2.2 视频生成

Wan 2.2 系列模型由阿里巴巴开发，支持 T2V（文本到视频）、I2V（图像到视频）和 S2V（声音驱动视频）。

## 一、模型概述

| 模型 | 参数量 | 功能 | 显存需求 |
|:----:|:------:|:----:|:--------:|
| Wan 2.2 5B T2V | 5B | 文本 → 视频 | 12-16GB fp8 |
| Wan 2.2 5B I2V | 5B | 图像 + 文本 → 视频 | 12-16GB fp8 |
| Wan 2.2 14B S2V | 14B | 声音/音频 → 驱动视频 | 32GB+ fp8 |

### 自定义节点

Wan 2.2 在 ComfyUI 0.3.0+ 有官方集成，或使用第三方节点：

```bash
cd ComfyUI/custom_nodes/
git clone https://github.com/kijai/ComfyUI-Wan-Superresolution.git
```

**更推荐使用 Workflow Templates**：`Workflow → Browse Workflow Templates` → 搜索 "Wan" → 选择 Wan 2.2 模板。

## 三、标准加载链

Wan 2.2 使用**分体加载**架构，不是传统的 Load Checkpoint：

```
UNETLoader (Wan2_2-T2V-5B) → ModelSamplingSD3
CLIPLoader (umt5_xxl) → CLIP Text Encode
VAELoader (Wan2_2-VAE) → VAE Decode
```

### 节点：ModelSamplingSD3

| 参数 | 推荐值 | 理由 |
|:----:|:------:|------|
| `shift` | 4.0 | 推荐 3.0-5.0，影响视频的动态幅度和稳定性 |

## 四、I2V 首帧过烤修复（LatentCut + LatentConcat）

Wan 2.2 I2V 模式常见**首帧过烤（overcooking）**——首帧与参考图像不符。

### 解决方案架构

```
Load Image → VAE Encode → LatentCut → LatentConcat → KSampler
EmptyLatentVideo ──────────────────→ LatentConcat ──┘
```

原理：显式将参考图像的潜空间编码结果与生成视频的潜空间拼接，消除过烤。

## 五、轻量路径：Lightning LoRA

| 场景 | 优点 | 缺点 |
|:----:|:----:|:----:|
| 快速原型/低显存（8-12GB） | 4 步即生成，显存降低约 30% | 质量和多样性略低 |

### 配置变更

在 `UNETLoader → ModelSamplingSD3 → KSampler` 链路中插入 LoRA：

| 参数 | 推荐值 |
|:----:|:------:|
| `lora_name` | `wan2.2-5b-lightning-lora.safetensors` |
| `strength_model` | 1.0 |
| KSampler `steps` | 4 |
| KSampler `cfg` | 1.0（推荐 1.0-2.0） |
| `sampler_name` | euler |
| `scheduler` | normal |
