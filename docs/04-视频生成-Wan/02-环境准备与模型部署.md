# Wan 2.2 环境准备与模型部署

> **前置**：已阅读 [Wan 核心概念与选型指南](01-Wan-核心概念与选型指南.md)，已确认使用的模型版本。

---

## 一、模型下载

```bash
export HF_ENDPOINT=https://hf-mirror.com
```

### 必要模型（文生视频 T2V）

| 模型文件 | 大小 | 存放路径 | 说明 |
|:---------|:----:|:---------|:------|
| `Wan2_2-T2V-5B-fp8.safetensors` | ~8GB | `models/checkpoints/` | UNET 核心模型（文生视频） |
| `Wan2_2-VAE.safetensors` | ~1.5GB | `models/vae/` | VAE 编码器 |
| `umt5_xxl_fp8.safetensors` | ~8GB | `models/clip/` | 文本编码器 |

### 选配模型

| 模型文件 | 大小 | 存放路径 | 说明 |
|:---------|:----:|:---------|:------|
| `Wan2_2-I2V-5B-fp8.safetensors` | ~8GB | `models/checkpoints/` | 图生视频专用 UNET |
| `wan2.2-5b-lightning-lora.safetensors` | ~500MB | `models/loras/` | Lightning 加速 LoRA |

> ⚠️ **重要**：Wan 的模型文件是**分体的**——UNET 放 `models/checkpoints/`，VAE 放 `models/vae/`，文本编码器放 `models/clip/`。不是一体文件，三个文件缺一不可。

## 三、验证 ComfyUI 版本

Wan 2.2 需要 **ComfyUI 0.3.0+**（内置 Wan 节点支持）：

```bash
cd /path/to/ComfyUI
git pull origin master
```

### 版本过低怎么办？

如果 `git pull` 显示已经是最新但版本仍是旧的：

```bash
cd /path/to/ComfyUI
git fetch --tags
git tag -l "v*" | tail -10
# 找到最高版本，然后：
git checkout v0.3.0
```

## 五、验证安装

1. **启动 ComfyUI**（或重启已有实例）
2. **右键搜索** `WanVideo`
3. 如果出现以下节点 → ✅ 安装成功：
   - `WanVideoTextToVideoSampler`（文生视频采样器）
   - `WanVideoImageToVideoSampler`（图生视频采样器）
   - `UNETLoader`（分体加载 UNET）
   - `ModelSamplingSD3`（采样调度器）

## 六、常见安装问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:------|
| 搜索不到 WanVideo 节点 | ComfyUI 版本 < 0.3.0 | `git pull origin master` + 重启 |
| UNETLoader 找不到模型 | 模型不在 `models/checkpoints/` | 手动下载并放入正确目录 |
| CLIPLoader 报错 | UMT5 模型放错位置 | 放入 `models/clip/` 而不是 `models/text_encoders/` |
| 启动后节点全红 | 模型文件缺失 | 检查三个目录的文件是否完整 |
| 下载慢 | 境外源 | `export HF_ENDPOINT=https://hf-mirror.com` |
