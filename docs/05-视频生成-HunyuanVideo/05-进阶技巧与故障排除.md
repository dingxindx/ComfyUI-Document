# HunyuanVideo 进阶技巧与故障排除

> **前置**：已掌握 HunyuanVideo 的基础工作流（T2V、I2V），理解节点参数的含义。本文涵盖 fp8 优化、蒸馏加速、常见问题排查和完整的检查清单。

---

## 一、fp8 优化

### 1.1 fp8 精度选择

HunyuanVideo-1.5 原生支持 fp8 推理，这是它相比 LTX（22B）和 Wan 14B 的核心优势之一。

| 精度 | 显存需求 (720×1280, 49帧) | 质量 | 推荐场景 |
|:----:|:-------------------------:|:----:|:---------|
| **fp8** | ~12GB | ✅ 良好 | **推荐**——大部分用户的甜点选择 |
| fp8 蒸馏版 | ~10GB | ✅ 良好 | **显存不足时首选**——8步出片 |
| bf16 | ~20GB+ | 🏆 最佳 | 追求极致质量（需 24GB 以上显存） |
| int8 / GGUF | ~8GB | ⚠️ 部分降低 | 极低显存方案（如可用） |

### 1.2 fp8 加载方式

使用 HunyuanVideoLoader 时设置精度为 fp8：

```text
HunyuanVideoLoader：
  model_name: hunyuan_video_720_fp8.safetensors
  precision: fp8  ← 关键
```

### 1.3 精度组合优化

如果 Kijai 节点支持独立设置各组件精度：

| 主模型精度 | VAE 精度 | 文本编码器精度 | 显存 | 质量 |
|:---------:|:---------:|:-------------:|:----:|:----:|
| fp8 | fp16 | fp16 | 低 | ✅ 平衡 |
| fp8 | fp8 | fp8 | **最低** | ⚠️ 略降 |
| bf16 | fp16 | fp16 | 高 | 🏆 最佳 |
| fp8 | fp16 | fp8 | 低 | ✅ 推荐组合 |

### 1.4 fp8 显存节省技巧

1. **先降帧数**：从 81 降到 49 可省 ~2GB 显存
2. **再降分辨率**：从 720×1280 降到 640×1152 可省 ~2GB
3. **使用蒸馏版本**：8步/12步替代 30步，显著减少计算量
4. **关闭其他进程**：浏览器标签、其他 AI 工具

## 二、快速蒸馏版本（8步/12步）

### 2.1 什么是 step-distilled 版本？

HunyuanVideo-1.5 提供了 **蒸馏（distilled）版本**，这是一个训练优化的快速推理版本：

| 版本 | 步数 | 生成速度 | 质量 |
|:----|:----:|:--------:|:----:|
| 标准版 | 30-50 步 | ~3-8min | 🏆 最佳（推荐高质量场景） |
| **蒸馏 12 步版** | **12 步** | ~1-2min | ✅ 良好（推荐日常使用） |
| **蒸馏 8 步版** | **8 步** | ~30s-1min | ✅ 可接受（推荐快速测试） |

### 2.2 使用蒸馏版本

**前提**：下载对应的蒸馏 checkpoint 文件。

```bash
set HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download tencent/HunyuanVideo-1.5 \
  hunyuan_video_720_distilled.safetensors \
  --local-dir models/checkpoints/
```

**工作流修改**：只需在 `HunyuanVideoLoader` 中选择蒸馏 checkpoint，然后调整采样器参数：

| 参数 | 标准版 (30步) | 蒸馏 12 步版 | 蒸馏 8 步版 |
|:-----|:-------------:|:------------:|:-----------:|
| steps | 30-50 | **12** | **8** |
| cfg | 6.0-7.0 | **5.0-6.0** | **4.0-5.0** |
| sampler_name | euler | euler | euler |
| 显存需求 | ~12GB | ~10GB | ~10GB |
| 生成时间 | ~3-8min | ~1-2min | ~30s-1min |

> ⚠️ **注意**：蒸馏版本对 cfg 更敏感。cfg 太高（>6.0）可能导致过饱和或运动僵硬，cfg 太低（<3.0）可能导致欠饱和。建议从 cfg=5.0 开始调试。

### 2.3 蒸馏版本 vs 标准版本使用建议

```text
显存 < 12GB 或需要快速迭代 → 蒸馏 8 步版 (s=8, cfg=4.5)
日常使用 → 蒸馏 12 步版 (s=12, cfg=5.5)
生成最终作品 → 标准 30 步版 (s=30, cfg=6.0)
追求极致质量 → 标准 50 步版 (s=50, cfg=7.0)
```

## 三、常见问题排查

### 节点问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:------|
| 🔴 **搜索不到 HunyuanVideo 节点** | 自定义节点未安装或未重启 | 安装 `ComfyUI-HunyuanVideoWrapper` 后重启 |
| 🔴 **HunyuanVideoLoader 找不到模型** | 模型不在正确目录 | 检查 `models/checkpoints/` 目录 |
| 🔴 **HunyuanVideoTextEncode 报错** | 缺少文本编码器文件 | 下载 CLIP-L 和 T5-XXL 到 `models/clip/` |
| 🔴 **节点全红** | 核心文件缺失 | 检查主模型、VAE、文本编码器三者是否齐全 |

### 生成质量问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:------|
| 🔴 **视频模糊** | steps 太少或 cfg 太低 | steps ≥ 30，cfg ≥ 5.0 |
| 🔴 **运动不连贯/闪烁** | 帧数太少或 steps 不够 | 增加帧数（≥49）或 steps（≥40） |
| 🔴 **画面全黑** | VAE 未连接或 VAE 文件错误 | 检查 VAELoader → VAEDecode 的连线 |
| 🔴 **画面过曝/过暗** | cfg 太高或太低 | 保持 cfg 在 5.0-7.0 之间 |
| 🔴 **内容不符合提示词** | 提示词不够具体 | 添加更多细节描述和运动描述 |
| 🔴 **中文提示词理解不准确** | 提示词太长稀释注意力 | 控制在 30-50 字以内 |

### I2V 问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:------|
| 🔴 **I2V 首帧过烤严重** | image_noise_scale 太高 | 降到 0.05-0.1 |
| 🔴 **I2V 首帧完全保留但后续闪烁** | image_noise_scale=0 | 提到 0.05-0.15 加一点噪声 |
| 🔴 **I2V 生成的视频和参考图内容不符** | 提示词描述了场景而非运动 | 提示词只写运动 |
| 🔴 **I2V 运行时报错 "image" 端口未连接** | VAE Encode 输出未连接 | 检查连线 |

### 显存/性能问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:------|
| 🔴 **OOM（显存不足）** | 分辨率或帧数太高 | 使用 fp8 + 降帧数（25帧）+ 降分辨率（640×1152） |
| 🔴 **生成时间过长** | 分辨率或 steps 太高 | 使用蒸馏版本（8步/12步） |
| 🔴 **I2V 比 T2V 更耗显存** | I2V 多一个 VAE Encode 步骤 | 先用 T2V 测试参数，再切 I2V |
| 🔴 **视频保存很大** | 帧数多+分辨率高 | 降低帧数或输出时压缩 |

### 帧数/分辨率问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:------|
| 🔴 **帧数报错** | length 不满足 8×N+1 | 选 9/17/25/33/49/65/81/97/129 等 |
| 🔴 **分辨率报错** | width/height 不是 32 的倍数 | 调整为 32 的倍数（704/736/768/800...） |
| 🔴 **生成结果比例不对** | 参考图比例与生成分辨率比例不匹配 | 保持比例一致 |

## 四、提示词优化技巧

### 4.1 结构化提示词

```text
正面：主体 + 场景 + 运动 + 画质 + 氛围
"女子, 东京夜景, 雨, 镜头缓缓左移, 电影光照, 忧郁氛围"
───────────────────────────────────────────────
负面：低质量 + 变形 + 瑕疵 + 水印
"模糊, 扭曲, 丑陋, 水印, 文字, 闪烁"
```

### 4.2 运动强度控制

| 运动描述 | 效果 |
|:---------|:------|
| "静止不动"、"保持" | 几乎没有运动 |
| "轻微晃动"、"缓缓飘动" | 温和的自然运动 |
| "散步"、"缓慢平移" | 中等运动（推荐） |
| "奔跑"、"快速旋转" | 剧烈运动（可能不稳定） |

### 4.3 使用 seed 复现

- 找到满意的输出后，记下 seed + steps + cfg 的组合
- 微调时只改一个参数，保持 seed 不变
- 这能帮你有针对性地调试

## 六、完整检查清单

在点击 Queue Prompt 前逐项确认：

### 环境与模型

- [ ] ComfyUI-HunyuanVideoWrapper 自定义节点已安装并重启
- [ ] 主模型文件在 `models/checkpoints/` 中
- [ ] VAE 文件在正确目录中
- [ ] 文本编码器文件（CLIP-L + T5-XXL）已下载
- [ ] 如使用国内网络，已设置 `HF_ENDPOINT=https://hf-mirror.com`

### T2V 工作流

- [ ] HunyuanVideoLoader 已加载正确的模型
- [ ] HunyuanVideoTextEncode 已填写正面和负面提示词
- [ ] HunyuanVideoVAELoader 已加载 VAE
- [ ] ModelSamplingSD3（如需要）已连接
- [ ] Empty Latent 的 width/height 是 **32 的倍数**
- [ ] Empty Latent 的帧数满足 **8×N+1**
- [ ] HunyuanVideoSampler 参数合理（steps 30+, cfg 6.0+）
- [ ] VAEDecode 已连接 VAE
- [ ] Save Video fps=24

### I2V 工作流

- [ ] 采样器使用 `HunyuanVideoI2VSampler`（不是 T2V 版）
- [ ] Load Image + VAE Encode 已添加
- [ ] VAE Encode 输出连接到 I2V 采样器的 `image` 端口
- [ ] image_noise_scale 在 0.05-0.2 之间
- [ ] 提示词只描述运动，不描述场景
- [ ] 参考图分辨率比例与生成分辨率一致

### 参数检查

- [ ] steps 设置合理（蒸馏版用 8/12，标准版用 30-50）
- [ ] cfg 在合理范围内（标准 5.0-7.0，蒸馏 4.0-6.0）
- [ ] 提示词包含明确的运动描述
- [ ] 帧数不是过高（首次测试建议 25-33）
- [ ] 显存足够（fp8 + 49帧 + 720×1280 ≈ 12GB）

### 通用检查

- [ ] 没有红色连线或红色节点
- [ ] 所有必要端口已正确连接
- [ ] 如果使用蒸馏模型，cfg 已相应降低
- [ ] 如果 I2V 首帧过烤，已降低 image_noise_scale
- [ ] 如果 OOM，已降低帧数或使用蒸馏版本
