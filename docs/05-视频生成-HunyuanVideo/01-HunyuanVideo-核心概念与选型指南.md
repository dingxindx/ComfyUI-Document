# HunyuanVideo 核心概念与选型指南

> **前置**：已掌握文生图基础操作，理解 CheckpointLoader 和 KSampler 等基本概念。
>
> **和 Wan 2.2 / LTX 2.3 最大的区别**：HunyuanVideo 是**腾讯开源**的视频生成模型，8.3B 参数，比 LTX（22B）轻量得多，但采用**分体加载**方式——主模型、文本编码器、VAE 各自独立加载。此外，HunyuanVideo 支持**双文本编码器**（CLIP-L + T5-XXL），提示词理解能力更强。

## 二、分体加载原理——HunyuanVideo 的核心架构

### 模型三要素

HunyuanVideo 采用**分体式加载**，由三个独立组件构成：

```
HunyuanVideo 的模型构成：
  ├── 主模型 (DiT / Diffusion Transformer) → HunyuanVideoLoader  → checkpoints/
  ├── 文本编码器 (CLIP-L + T5-XXL)         → CLIPLoader         → clip/ 或 text_encoders/
  └── 3D VAE (视频编解码器)                → VAELoader          → vae/ 或 hunyuanvideo/
```

### 分体加载的好处

1. **精度灵活**：主模型用 fp8、VAE 用 fp16，各取所需
2. **升级灵活**：更新模型版本只需换主模型文件，无需重下全套
3. **调试方便**：哪个组件出问题一目了然

### 分体加载需要注意

1. 需要多个加载器节点（比 LTX 的一体加载多几步操作）
2. 文本编码器**有两个**（CLIP-L + T5-XXL），需要确认是否逐个加载
3. 不同版本的文件目录可能不同，以 Kijai 节点要求为准

## 三、HunyuanVideo 的帧数与分辨率规则

### 帧数规则：8×N+1

HunyuanVideo 的帧数规则与 LTX 相同，必须满足 **`length = 8×N+1`**（N 为任意正整数）。

| length | 8×N+1 | 时长 @24fps |
|:------:|:-----:|:-----------:|
| 9 | 8×1+1 | ~0.4s |
| 17 | 8×2+1 | ~0.7s |
| **25** | **8×3+1** | **~1.0s** |
| 33 | 8×4+1 | ~1.4s |
| **49** | **8×6+1** | **~2.0s** |
| 65 | 8×8+1 | ~2.7s |
| **81** | **8×10+1** | **~3.4s** |
| 97 | 8×12+1 | ~4.0s |
| 129 | 8×16+1 | ~5.4s |

> 📌 **推荐值**：
> - **快速测试**：25-33 帧（1-1.4s）
> - **标准生成**：49-81 帧（2-3.4s）
> - **质量优先**：97 帧（4s）
> - **显存不足时**：先降帧数而不是降分辨率

### 分辨率规则：32 的倍数

HunyuanVideo 的 width/height 必须是 **32 的倍数**（与 LTX 相同，比 Wan 的 16 倍数更严格）。

| 推荐分辨率 | 说明 |
|:----------:|:------|
| **720×1280** | ✅ **原生分辨率**（16:9 竖屏，推荐） |
| **1280×720** | ✅ 横屏，与原生同尺寸 |
| 640×1152 | ⚠️ 降级使用（显存不足时） |
| 576×1024 | ⚠️ 降级使用 |

> 💡 注意：HunyuanVideo 的原生分辨率是 **720×1280（竖屏）**，与很多视频模型相反。如果生成横屏视频，使用 1280×720。

## 四、HunyuanVideo vs LTX vs Wan 选型指南

| 场景 | 推荐模型 | 理由 |
|:-----|:---------|:------|
| 🥇 **追求质量与速度平衡** | **HunyuanVideo-1.5** | 8.3B 轻量、fp8 省显存、质量好 |
| ⚡ **极限速度** | **HunyuanVideo-1.5 distilled** | 8步出片，秒级生成 |
| 🏠 **中文提示词最佳** | **HunyuanVideo-1.5** | 腾讯出品，中英文支持最佳 |
| 💾 **显存受限 (≤16GB)** | **HunyuanVideo-1.5 fp8** | 12GB 可跑，低于 LTX 和 Wan 14B |
| 🎨 **图生视频** | **HunyuanVideo-1.5 I2V** | v1.5 原生支持 I2V |
| 🎯 **长视频 (5s+)** | LTX 2.3 | 支持更长帧序列 |
| 🧪 **快速迭代测试** | Wan 2.2 5B + Lightning LoRA | 4步 LoRA 极速 |
| 🖥️ **极致质量** | Wan 2.2 14B（显存充足时） | 参数量更大，细节更丰富 |

### 一句话总结

> **显存有限、要求中文支持好、想要图生视频 → 选 HunyuanVideo-1.5**
>
> **追求极速（4步）/ 最大化质量 → 选 Wan 2.2（5B 或 14B）**
>
> **需要超长视频 / 一体式简单加载 → 选 LTX 2.3**
