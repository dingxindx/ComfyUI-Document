# HunyuanVideo 环境准备与模型部署

> **前置**：已阅读 [HunyuanVideo 核心概念与选型指南](01-HunyuanVideo-核心概念与选型指南.md)，已确认使用的模型版本（推荐 v1.5）。
>
> **与 Wan/LTX 的差异**：HunyuanVideo 需要通过 **Kijai 的自定义节点**（ComfyUI-HunyuanVideoWrapper）使用，不是 ComfyUI 内置节点。需要先安装自定义节点，再下载模型文件。

---

## 一、安装自定义节点

### 方法 1：ComfyUI Manager 安装（推荐）

1. 打开 ComfyUI → 点击 **Manager**
2. 搜索 → **ComfyUI-HunyuanVideoWrapper**
3. 作者：**kijai** → 点击 Install
4. 重启 ComfyUI

### 方法 2：Git Clone 手动安装

```bash
cd /path/to/ComfyUI/custom_nodes/
git clone https://github.com/kijai/ComfyUI-HunyuanVideoWrapper.git
pip install -r ComfyUI-HunyuanVideoWrapper/requirements.txt
```

> **国内镜像加速**：
> ```bash
> git clone https://gitclone.com/github.com/kijai/ComfyUI-HunyuanVideoWrapper.git
> # 或使用 ghproxy
> git clone https://ghproxy.com/https://github.com/kijai/ComfyUI-HunyuanVideoWrapper.git
> ```

### 依赖安装

```bash
cd ComfyUI-HunyuanVideoWrapper/
pip install -r requirements.txt
```

> 如果安装过程中下载 `hunyuan_video_t2v` 或相关依赖包下载慢，可设置镜像源：
> ```bash
> pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
> ```

## 二、模型下载清单

### 设置镜像源（国内用户必做）

```bash
export HF_ENDPOINT=https://hf-mirror.com
```

### 必要模型

| 组件 | 模型文件 | 大小 | 存放路径 | 说明 |
|:----|:---------|:----:|:---------|:------|
| **主模型 (DiT)** | `hunyuan_video_720_fp8.safetensors` | ~8.5GB | `models/checkpoints/` | 核心 UNET/Transformer 模型 |
| **文本编码器** | CLIP-L + T5-XXL 相关文件 | ~10GB+ | `models/clip/` 或 `models/text_encoders/` | 双编码器组合 |
| **3D VAE** | `hunyuan_video_vae.safetensors` | ~1.5GB | `models/vae/` 或 `models/hunyuanvideo/` | 视频编解码 |

### 可选模型

| 模型文件 | 大小 | 存放路径 | 说明 |
|:---------|:----:|:---------|:------|
| `hunyuan_video_720_distilled.safetensors` | ~8.5GB | `models/checkpoints/` | 蒸馏版本（8步/12步快速推理） |
| `hunyuan_video_720_bf16.safetensors` | ~16GB | `models/checkpoints/` | bf16 精度版本（质量更好但更耗显存） |

### 下载命令参考

```bash
# 设置镜像
export HF_ENDPOINT=https://hf-mirror.com

# 下载主模型（fp8 版本）
huggingface-cli download tencent/HunyuanVideo-1.5 \
  hunyuan_video_720_fp8.safetensors \
  --local-dir models/checkpoints/

# 下载 VAE
huggingface-cli download tencent/HunyuanVideo-1.5 \
  hunyuan_video_vae.safetensors \
  --local-dir models/vae/

# 或直接下载整个仓库（需要大量磁盘空间）
huggingface-cli download tencent/HunyuanVideo-1.5 \
  --local-dir models/hunyuanvideo/
```

> ⚠️ **注意**：HunyuanVideo-1.5 在 HuggingFace 上的仓库结构可能包含多个子文件夹。请根据实际安装的 Kijai 节点版本确认具体所需的文件路径。

## 三、目录结构

安装完成后，ComfyUI 的 `models/` 目录应类似如下结构：

```text
ComfyUI/models/
├── checkpoints/
│   ├── hunyuan_video_720_fp8.safetensors    (必选，主模型)
│   └── hunyuan_video_720_distilled.safetensors (可选，蒸馏版)
├── clip/
│   ├── clip_l.safetensors                    (CLIP-L 文本编码器，如有)
│   ├── t5_xxl.safetensors                    (T5-XXL 文本编码器，如有)
│   └── ...其他文本编码器文件
├── vae/
│   └── hunyuan_video_vae.safetensors         (必选，VAE)
└── hunyuanvideo/                             (可选，按需)
    └── ...
```

> 📌 **目录说明**：
> - 具体文件存放路径**以 Kijai 节点实际要求为准**，安装后打开 ComfyUI，右键添加节点，查看各加载器的文件下拉列表中显示哪些路径。
> - 一些版本的 Kijai 节点可能要求将 VAE 放在 `models/hunyuanvideo/` 下，请按节点提示操作。

## 四、验证安装

1. **重启 ComfyUI**（或刷新节点列表）
2. **右键搜索**以下节点，确认都已出现：

| 节点名 | 说明 |
|:-------|:------|
| `HunyuanVideoLoader` | 加载主模型（DiT / UNET） |
| `HunyuanVideoTextEncode` | 双文本编码器（CLIP-L + T5-XXL 提示词编码） |
| `HunyuanVideoVAELoader` | 加载 VAE（视频编解码） |
| `HunyuanVideoSampler` | 文生视频采样器 |
| `HunyuanVideoI2VSampler` | 图生视频采样器（v1.5 支持） |

> 📌 **节点名确认**：以上节点名基于常见命名惯例，**请以实际安装的 Kijai 节点版本为准**。如果搜索不到，请检查：
> 1. 自定义节点是否安装成功
> 2. 是否重启了 ComfyUI
> 3. 是否有红色报错提示依赖缺失

### 快速烟雾测试

安装后可以快速验证是否一切正常：

1. 添加 `HunyuanVideoLoader` → 检查文件下拉列表中是否有 `hunyuan_video_720_fp8`
2. 添加 `HunyuanVideoTextEncode` → 确认不会报 red node
3. 添加 `HunyuanVideoVAELoader` → 确认 VAE 文件可加载
4. 添加 `HunyuanVideoSampler` → 确认参数可设置

如果以上四个节点都正常 → ✅ 安装成功！

## 五、常见安装问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:------|
| 搜索不到 HunyuanVideo 节点 | 自定义节点未安装或未重启 | `git clone` 后重启 ComfyUI |
| HunyuanVideoLoader 找不到模型 | 模型未下载或路径不对 | 检查 `models/checkpoints/` 目录 |
| 提示 "CUDA out of memory" | 显存不足 | 使用 fp8 模型 + 降低分辨率/帧数 |
| 中文提示词乱码 | 编码问题 | 使用 Unicode 编码保存工作流 |
| 下载慢 | 境外源 | `export HF_ENDPOINT=https://hf-mirror.com` |
| 安装依赖报错 | 缺少编译工具或 Python 版本不兼容 | 确认 Python ≥ 3.10，安装 build tools |
| 节点全红 | 模型文件缺失 | 检查三个核心文件是否都已下载 |
