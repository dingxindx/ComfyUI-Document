# 工作流学习三步法

当你想实现一个具体效果（如「生成一张保持人物一致的换装照」），按以下三步走。

## 第一步：输出决定输入——反推需要的「能力块」

先**不要**想节点或模型，而是问：**"最终画面里，什么是不可妥协的特征？"**

| 核心要求 | 必须的能力模块 | 对应模型/节点族 |
|:--------:|:--------------:|:---------------:|
| 固定角色脸/物 | 身份保持 | IPAdapter / InstantID / FaceID / PuLID |
| 特定画风 | 风格控制 | LoRA（风格类）/ Style Model / 微调大模型 |
| 精确姿态/构图 | 骨架或深度控制 | ControlNet（OpenPose / Depth / Canny） |
| 多元素一致合并 | 区域融合 | Attention Mask / Latent Composite / LayerDiffuse |
| 背景换掉但主体自然 | 重绘或抠图 | Inpaint / SAM / RemoveBG |

> 💡 **练习**：列 5 个你想做的工作流，强行填这张表。卡住了再去搜对应关键词。

## 第二步：节点 = 能力模块的「标准插头」

不要背节点，按功能类别存路径：

| 能力模块 | 常用节点 | 替代方案 |
|:--------:|:--------:|:--------:|
| 提示词增强 | CLIP Text Encode (Prompt) | 无替代 |
| ControlNet 预处理 | Auxiliary Preprocessors | OpenPose→DW Pose Detector |
| 身份保持 | IPAdapter Unified Loader + Apply | InstantID（内存占用小）|
| 分区域生成 | CLIPSetMask / Detailer (Segs) | LayerDiffuse |
| 重绘 | VAE Encode (for Inpaint) + Apply ControlNet (inpaint) | 直接使用 |

**关键心法**：
- 90% 的工作流只需要 10 个核心节点类
- 学会 **「官方范例工作流」的节点连线颜色**（紫色=模型，蓝色=潜空间，橙色=条件），比记名字快 3 倍

## 第三步：模型选择——「型号与参数」而非「名字」

| 用途 | 推荐模型 | 特点 |
|:----:|:--------:|:----:|
| 写实人像 | RealisticVision / MajicMix Realistic | 稳定通吃 |
| 二次元 | Anything V5 / Counterfeit | 动漫首选 |
| 2.5D 混合 | DreamShaper | 全能型 |
| ControlNet | 与大模型配套的 `.safetensors` | 控图专用 |

**模型搜索技巧**：
1. 去 Civitai 或 哩布哩布 搜你想做的**效果图的关键词**
2. 看该图的 **"Recipe"** 区域，会直接列出：Base Model + LoRA + CFG + Steps
3. **复制那套组合**，不要自己猜

## 速查表

| 问题 | 推荐做法 |
|:----:|:--------:|
| 不知道用什么大模型 | 去 C 站搜同效果图，看它的 base model |
| 不确定要加 ControlNet 还是 LoRA | 固定布局用 ControlNet，固定画风/物件用 LoRA |
| 要换脸但不想训练 | 直接用 IPAdapter |
| 生成图主体错位 | 把 latent 和 mask 分别送到 KSampler 的 denoise 处 |
| 多个 ControlNet 怎么组合 | 用 Control Net Stack 节点，权重从 0.4 开始叠 |
