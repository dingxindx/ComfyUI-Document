# 故障排查手册——你的红框救援指南

> 遇到错误时，按照"看现象 → 看原因 → 看解决"三步定位。如果一种方案解决不了，接着试下一种。

## 一、节点变红排查树

```
节点变红了
│
├─ 所有节点都红？
│   ├─ ✅ 没下载任何模型 → 去 00-03 下载第一个模型
│   └─ ✅ 模型目录没挂载 → 刷新 ComfyUI（F5 或重启）
│
├─ 只有某个节点红？
│   ├─ 是 CheckpointLoaderSimple？
│   │   ├─ 下拉框里没有模型 → 检查 models/checkpoints/ 目录是否有 .safetensors 文件
│   │   └─ 下拉框有模型但选了还红 → 刷新 ComfyUI
│   │
│   ├─ 是 CLIP Text Encode？
│   │   └─ 检查 CheckpointLoaderSimple 的 CLIP 端口是否连接到了它
│   │
│   ├─ 是 VAE Decode？
│   │   └─ 检查 CheckpointLoaderSimple 的 VAE 端口是否连接到了它
│   │
│   ├─ 是 ControlNet 相关的节点？
│   │   ├─ ControlNetLoader 下拉空白 → 检查 models/controlnet/ 目录是否有 .pth/.safetensors
│   │   └─ 预处理器报错 → 缺少模型文件，检查 models/aux_preprocessors/
│   │
│   ├─ 是 LTX/Van 视频节点？
│   │   └─ 确认已安装对应的 custom_nodes（ComfyUI-LTXVideo 等）
│   │
│   └─ 是自定义节点（custom node）？
│       ├─ 目录内有 requirements.txt → 手动 pip install -r requirements.txt
│       └─ 节点与 ComfyUI 版本不兼容 → 尝试更新 ComfyUI 或该节点
│
└─ 刷新后新节点变红？
    └─ 模型文件刚下载完成，ComfyUI 还没重新扫描 → 重启 ComfyUI
```

## 二、Queue Prompt 点了没反应

| 现象 | 原因 | 解决 |
|:-----|:-----|:-----|
| 按钮点了，但什么也没发生 | 终端有红色错误信息 | 去终端窗口看最后几行文本 |
| 按钮点了，进度条不动 | 某个节点参数没设置 | 检查每个节点的输入框是否为空 |
| 按钮点了，弹出 JS 报错 | 浏览器缓存问题 | 按 F12 打开控制台看错误 |
| 点击 Queue Prompt 跳到下一个工作流 | 当前是队列模式 | 检查 Queue 里是否已经有任务 |

**去终端看错误**：ComfyUI 的终端（你运行 `python main.py` 的那个窗口）会打印详细的错误信息。把最后 10-20 行贴出来，基本就能定位问题。

## 三、生成了但图片有问题

### 全黑/全绿

```
生成全黑或全绿
│
├─ 全黑
│   ├─ VAE 没接上 → 检查 CheckpointLoaderSimple 的 VAE 端口到 VAE Decode 的黄色连线
│   └─ 模型下载损坏 → 重新下载模型
│
├─ 全绿/全彩色块
│   └─ VAE 与模型不兼容 → 使用模型自带的 VAE（用 CheckpointLoaderSimple）
│
├─ 全灰/噪点/雪花
│   └─ steps 太低或 cfg 过高 → 把 steps 设为 20-30，cfg 设为 7-9
│
└─ 一片模糊
    ├─ 分辨率与模型不匹配 → SD1.5 用 512×512，SDXL 用 1024×1024
    └─ denoise 太低 → 文生图用 denoise=1.0
```

### 颜色/构图问题

| 现象 | 原因 | 解决 |
|:-----|:-----|:-----|
| 颜色过饱和/失真 | cfg 太高 | 降到 7-9 |
| 颜色太淡/不遵循提示 | cfg 太低 | 升到 7-9 |
| 构图重复/画面复制 | 采样器选错了 | 换 `euler` 或 `dpmpp_2m` |
| 出现多余的人和物体 | 负面提示词不够 | 添加 `extra person, extra object, duplicate` |
| 面部扭曲/手指畸形 | 模型本身问题 | 加负面词 `bad hands, bad face` 或换模型 |

## 四、安装与启动问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:-----|
| `git clone` 下载到一半卡住 | 网络问题 | 按 Ctrl+C 重试，或换 `gitclone.com` 镜像 |
| `pip install` 报红色错误 | 网络超时/依赖冲突 | 1️⃣ `pip install -i https://pypi.tuna.tsinghua.edu.cn/simple ...` 2️⃣ 从头创建新虚拟环境 |
| `pip install` 提示找不到版本 | Python 版本不兼容 | 确保 Python 3.10-3.12，不要用 3.13 |
| `python main.py` 报 `ModuleNotFoundError` | 依赖没装全 | `pip install -r requirements.txt` |
| `python main.py` 报端口被占用 | 另一个 ComfyUI 实例在运行 | `python main.py --port 8189` |
| 浏览器访问 `127.0.0.1:8188` 显示"无法访问" | ComfyUI 没启动 | 检查终端是否还在运行，是否有"Prompt server running on" |
| 访问 `localhost:8188` 不行，`127.0.0.1:8188` 可以 | hosts 文件问题 | 直接用 `127.0.0.1:8188` |
| 终端出现 `Killed: 9` | 内存不足被系统杀进程 | 换小模型或加 `--novram` 启动 |

## 五、模型相关

| 问题 | 原因 | 解决 |
|:-----|:-----|:-----|
| 下拉框找不到已下载的模型 | 没刷新/放错目录 | 1️⃣ F5 刷新 2️⃣ 模型放 `models/checkpoints/` |
| 模型下拉框有文件但选了还是红 | 文件损坏或不完整 | 删除后重新下载 |
| 下载的模型是 `.ckpt` 不是 `.safetensors` | 旧格式 | 两者都可用，`.safetensors` 更安全更快 |
| 模型下载中途断了 | 网络不稳 | 删除不完整的文件再重下 |
| LoRA 没起作用 | 没写触发词 | 去 Civitai 查看 LoRA 页面的 trigger words |
| LoRA 导致画面崩溃 | 权重太高或基座模型不匹配 | 降低 strength_model 到 0.5 以下 |
| 多个 LoRA 互相干扰 | 总权重超过 2.0 | 保持三个 LoRA 总 strength 不超过 2.0 |

## 六、视频生成常见问题

| 问题 | 原因 | 解决 |
|:-----|:-----|:-----|
| 视频生成 OOM（内存不足） | 分辨率太高/帧数太多 | 降低分辨率（640×384）、减少帧数（33帧） |
| 视频全黑 | VAE 没接上 | 检查 VAE 黄色线连接 |
| 视频运动不连贯 | steps 太少或 cfg 太高 | steps ≥ 40，cfg ≤ 4.0 |
| 首帧被过度修改 | image_noise_scale 太高 | 图生视频降到 0.1-0.2 |
| 两阶段生成结果不如单阶段 | denoise 设置不对 | 阶段二 denoise 设为 0.4-0.6 |
| LTX 模型加载超时 | Gemma 编码器太大 | 换 fp8 版或使用 GGUF 量化 |

## 七、万能排查流程

如果上面的分类没覆盖你的问题，按这个流程走：

```
1️⃣ 看终端输出 → 最后 10-20 行，通常有错误原因
   ├─ "No module named xxx" → pip install xxx
   ├─ "CUDA out of memory" → 换小模型/降低分辨率
   ├─ "File not found" → 检查文件路径
   └─ "Cannot connect to" → 网络问题，检查代理/镜像

2️⃣ 检查节点 → 哪个节点是红色的？
   ├─ 红 ＝ 缺少模型/依赖
   └─ 红色连线 ＝ 端口类型不匹配

3️⃣ 重启大法
   ├─ 刷新浏览器（F5）
   ├─ 重启 ComfyUI（Ctrl+C → python main.py）
   └─ 重启电脑（有奇效）

4️⃣ 换模型
   └─ 如果某个模型的参数不对，换另一个 Checkpoint 试试

5️⃣ 搜索引擎
   └─ 把错误信息复制到百度/谷歌搜索
```

## 八、错误信息截图准则

当需要求助时，**不要只拍屏幕**，请提供：
1. 终端窗口中错误信息最后的 10-20 行文字（**复制文本，不要截图**）
2. 当前工作流的截图/描述
3. 你想达到什么效果

> **求助话术模板**：  
> "我在 ComfyUI 跑文生图，加载了 DreamShaper 8，steps=20，cfg=7，点击 Queue Prompt 后终端报 [粘贴错误信息]。模型放在 models/checkpoints/ 下。请问是什么问题？"
