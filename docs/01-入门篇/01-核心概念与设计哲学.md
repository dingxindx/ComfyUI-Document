# 核心概念与设计哲学

## 一、ComfyUI 与 WebUI 的本质区别

ComfyUI 与 AUTOMATIC1111/WebUI 的区别在于三个核心设计决策：

### 1. 潜空间（Latent Space）—— 一切生成的中枢

Stable Diffusion 不在像素空间（RGB 像素）中做计算，而是在**压缩了 48 倍的潜空间**中工作。流程如下：

```
像素图 → VAE编码器 → 潜空间张量 → UNet 去噪 → VAE解码器 → 像素图
```

> 💡 理解潜空间意味着你明白为什么 VAE 节点（VAELoader / VAEEncode / VAEDecode）是任何工作流的起点和终点。VAE 是连接像素世界和潜空间的桥梁。

### 2. 节点数据流 —— 有向无环图（DAG）

ComfyUI 的每个节点是一个纯函数：接收输入张量 → 处理 → 输出张量。数据沿着连线单向流动，**不允许循环**。

关键推论：
- **执行顺序由依赖关系决定**，而非画布上的位置
- 一个输出可以分叉给多个节点（数据复用）
- 同一个输入端口只能接收一根连线

### 3. 队列机制 —— 批处理与中断

| 功能 | 说明 |
|------|------|
| **Queue Prompt** | 将当前工作流加入执行队列 |
| **Queue Prompt (Batch)** | 批量生成 N 张，自动随机种子 |
| **Interrupt（中断）** | 立即停止当前生成，结果保留已完成的步数 |

> 💡 SSD/HDD 空间是隐性瓶颈。每个工作流会在 `output/` 目录生成文件，长时间运行视频生成注意磁盘空间。

## 二、节点连接颜色约定

ComfyUI 使用不同颜色的连线直观区分数据类型：

| 颜色 | 数据类型 | 说明 |
|:----:|----------|------|
| **紫色** | MODEL | 扩散模型（Checkpoint、LoRA） |
| **蓝色** | LATENT | 潜空间张量 |
| **橙色** | CONDITIONING | 条件（提示词编码） |
| **粉色** | CLIP | 文本编码器 |
| **绿色** | IMAGE | 像素图像 |
| **黄色** | VAE | VAE 模型 |
| **红色** | MASK | 遮罩（用于重绘等） |
| **灰色** | CONTROL_NET | ControlNet 模型 |
| **青色** | AUDIO | 音频数据 |

> 💡 学会看颜色比记节点名快 3 倍。90% 的工作流只需要约 10 个核心节点类。

## 三、ComfyUI 的精髓

> *"ComfyUI 的精髓不在于记住了多少节点，而在于理解了数据如何在潜空间中流动。"*
