# 图生视频工作流（LTX 2.3）

从一张静态图片生成延续运动的视频。

## 一、与文生视频的差异

- 用 `LTXVImageToVideoSampler` 替代 `LTXVTextToVideoSampler`
- 增加 `LoadImage` + `ImageResize` 节点作为首帧输入
- 运动由「提示词」驱动，而非「从零生成」

## 二、核心架构

```
CheckpointLoaderSimple ─┬─→ LTXVImageToVideoSampler → VAEDecodeTiled → SaveVideo
LTXTextEncoderLoader ───┤
LoadImage → ImageResize ─┘ (首帧图像)
EmptyLTXVLatentVideo ─────→ (潜空间)
CLIP Text Encode ─────────→ (提示词)
```

## 三、新增节点

### 节点：LoadImage + ImageResize

| 项目 | 说明 |
|------|------|
| **参数** | `width`/`height`: 与 EmptyLatent 一致（如 832×480）|
| **resampling** | lanczos |
| **理由** | 首帧分辨率应与目标视频分辨率一致或成比例 |

### 节点：LTXVImageToVideoSampler

| 参数 | 推荐值 | 理由 |
|:----:|:------:|------|
| `steps` | 40-60 | 图生视频需更多步数以平滑首帧过渡 |
| `cfg` | 3.0-3.5 | 过高会扭曲原图 |
| `image_noise_scale` | 0.1-0.3 | 控制首帧保留程度；越低越接近原图 |

## 四、图生视频的提示词技巧

提示词应描述**运动**而非场景：

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| "a beautiful mountain landscape" | "camera slowly pans left, clouds drift across the sky, light rays through trees" |
| "a woman smiling" | "the woman slowly turns her head, hair blown by gentle wind, subtle smile" |
