# LTX 2.3 核心概念与设计哲学

## 一、模型概述

LTX 2.3 是 Lightricks 开源的视频生成模型，参数量 22B。

| 特性 | LTX 2 | LTX 2.3 |
|------|:-----:|:--------:|
| 参数量 | 19B | 22B |
| 竖屏支持 | 一般 | 大幅优化（9:16）|
| 音频质量 | 有杂音 | 更清晰 |
| 图生视频 | 偶有卡顿 | 运动更一致 |
| 文字渲染 | 模糊 | 更清晰 |

## 二、三个核心设计哲学

### 1. 两阶段生成哲学

LTX 2.3 官方推荐**不在单次采样中追求最终质量**，而是：
- **第一阶段**：半分辨率生成，确定构图和运动轨迹（成本低）
- **第二阶段**：潜空间放大，补充细节（质量高）

本质上是**空间解耦**——将「做什么」和「做多细」分离。

### 2. 潜空间放大的优势

传统做法：生成低分辨率视频 → 解码为像素 → 用 ESRGAN 等模型放大 → **丢失时序一致性**。

LTX 2.3 的做法：在潜空间中直接放大（`LTXVLatentUpsampler`），保持运动连贯性的同时增加空间细节。

### 3. 文本编码器的选择逻辑

**Gemma 3 12B** 是 LTX 2.3 指定的文本编码器，不可替换为 CLIP 或其他模型。因为 LTX 2.3 的训练 pipeline 与 Gemma 3 的 embedding space 深度绑定。

⚠️ 如果显存不足（<24GB），可使用 fp8 或 GGUF Q4 量化版本的 Gemma 3。

## 三、Checkpoint 类型选择

| Checkpoint 类型 | 显存占用 | 质量 | 推荐场景 |
|:---------------:|:--------:|:----:|---------|
| **dev (BF16)** | ~44GB | 最高 | 24GB+ 显卡 |
| **fp8** | ~23-29GB | 接近 dev | 16-24GB 显卡，日常首选 |
| **distilled** | ~23GB | 略低于 fp8 | 快速迭代 + 搭配 distilled LoRA |

> 💡 distilled checkpoint 必须配合 `ltx-2.3-22b-distilled-lora-384` 使用，且 CFG 需设为 1.0，步数 4-8 步。

## 四、潜空间约束

| 参数 | 规则 | 说明 |
|------|:----:|------|
| width/height | **32 的倍数** | 模型步幅约束 |
| length（帧数） | **8×N+1** | 如 33、41、49、57...201 |

> 💡 常见帧数：`length=33`（~1.1 秒 @30fps）、`length=49`（~2 秒）、`length=201`（~8.3 秒）
