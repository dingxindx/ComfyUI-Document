# 高级优化与故障排除（LTX 2.3）

## 一、显存优化策略矩阵

| 显存 | 推荐方案 | 关键设置 |
|:---:|:--------:|---------|
| 12GB | GGUF Q3 + VAE Offload | 启用 `--novram` 启动参数 |
| 16GB | GGUF Q4 或 fp8 | 两阶段工作流，阶段一半分辨率 |
| 24GB | fp8 或 BF16 | 两阶段工作流 + 全分辨率 |
| 32GB+ | BF16 | 批量生成，高分辨率 |

### VAE Offload

```bash
python main.py --novram
```

或在节点层面：使用 `VAEDecodeTiled` 而非 `VAEDecode`，保持 `tile_size=512`。

## 二、RTX 视频超分辨率升频（可选）

使用 RTX 显卡硬件加速将生成视频升频至 4K。

| 项目 | 说明 |
|:----:|------|
| **安装** | ComfyUI Manager → 搜索 "RTX Video Super Resolution" |
| **位置** | VAEDecode 之后、SaveVideo 之前 |
| **参数** | `scale`: 2（1080p→4K）或 3（720p→4K）；`quality`: ULTRA |

## 三、首帧过烤问题修复

使用图生视频时，有时首帧会被过度修改（"过烤"）。解决方案：使用 `LatentCut` 和 `LatentConcat` 节点将首帧潜空间与生成潜空间拼接。

核心思路：
1. 从 EmptyLatent 获取首帧潜空间
2. 使用 LatentCut 提取首帧
3. 使用 LatentConcat 将首帧与后续生成帧拼接
4. 确保首帧在采样过程中保持固定

## 四、常见错误及解决方案

| 错误现象 | 原因 | 解决方案 |
|:--------:|:----:|---------|
| `Value not in list` | 模型下拉框未找到模型 | 检查模型是否放在正确目录，刷新节点列表 |
| `Missing Node Type`（红色节点）| 自定义节点未安装 | 安装 ComfyUI-LTXVideo |
| OOM during decode | VAE 解码显存不足 | 使用 VAE Decode Tiled，降低 tile_size |
| Gemma 编码器崩溃 | 文件路径错误或损坏 | 确认文件在 `text_encoders/`，重新下载 |
| 视频全黑/全绿 | VAE 连接错误 | 确保 VAE 端口正确连接 |
| 运动不连贯/抖动 | steps 太少或 cfg 过高 | steps 增加到 40+，cfg 降低到 3.0-3.5 |
| 首帧被过度修改 | image_noise_scale 过高 | 降低到 0.1-0.2 |

## 五、质量优化检查清单

- [ ] 分辨率宽高是 32 的倍数？
- [ ] length 满足 8×N+1？
- [ ] steps 在推荐范围内？（文生视频 30-50，图生视频 40-60）
- [ ] cfg 在 2.0-5.0 之间？
- [ ] 提示词具体描述了运动/场景？
- [ ] 高质量要求时使用了双阶段工作流？
- [ ] 显存充足？（检查是否触发了显存交换）
