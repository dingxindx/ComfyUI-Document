# LTX 2.3 高级优化与故障排除

## 一、显存优化策略

| 显存 | 推荐方案 |
|:----:|:---------|
| 12GB | GGUF Q3 + VAE Offload（启动加 `--novram`）|
| 16GB | GGUF Q4 或 fp8，两阶段先用半分辨率 |
| 24GB | fp8 或 BF16 |
| 32GB+ | BF16，批量生成 |

VAE Offload：`python main.py --novram` 或使用 `VAEDecodeTiled` 保持 `tile_size=512`。

## 二、首帧过烤问题修复

用图生视频时，首帧被 AI 过度修改（"过烤"）。解决办法：

使用 `LatentCut` + `LatentConcat` 节点将参考图像的潜空间与生成的视频潜空间拼接。

**原理**：显式提供精确的首帧潜空间信号，避免模型"脑补"首帧。

## 三、常见错误

| 错误现象 | 原因 | 解决 |
|:---------|:-----|:-----|
| `Missing Node Type` | 缺少 ComfyUI-LTXVideo | 安装后重启 |
| `Value not in list` | 模型下拉找不到文件 | 检查 models/ 目录是否正确 |
| OOM during decode | VAE 解码显存不足 | 用 VAE Decode Tiled，降低 tile_size |
| 视频全黑 | VAE 端口没连接 | 检查 VAE 黄色线 |
| 运动不连贯 | steps 太少或 cfg 太高 | steps ≥ 40，cfg ≤ 4.0 |
| 首帧被过度修改 | image_noise_scale 太高 | 降到 0.1-0.2 |

## 四、质量检查清单

- [ ] 分辨率宽高是 32 的倍数？

- [ ] 高质量需求用了两阶段工作流？
