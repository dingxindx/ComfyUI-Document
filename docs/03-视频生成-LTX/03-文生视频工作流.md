# 文生视频基础工作流（LTX 2.3）

这是理解 LTX 2.3 工作流结构的**最小单元**。后续所有工作流都在此基础上扩展。

## 一、整体结构

```
CheckpointLoaderSimple ─┬─→ LTXVTextToVideoSampler → VAEDecodeTiled → SaveVideo
LTXTextEncoderLoader ───┘
EmptyLTXVLatentVideo ─────→ (潜空间初始化)
CLIP Text Encode (Positive) ─→
CLIP Text Encode (Negative) ─→
```

## 二、逐节点搭建

### 节点 1：CheckpointLoaderSimple

| 项目 | 说明 |
|------|------|
| **添加方式** | 右键 → 搜索 "checkpoint" |
| **关键参数** | `ckpt_name`: 选择 `ltx-2.3-22b-dev-fp8.safetensors` |
| **输出连接** | `MODEL` → LTXVTextToVideoSampler; `VAE` → VAEDecodeTiled |

> ⚠️ LTX 2.3 的 VAE 已集成在 checkpoint 中，无需单独加载 VAE 文件。

### 节点 2：LTXTextEncoderLoader（Gemma 文本编码器）

| 项目 | 说明 |
|------|------|
| **添加方式** | 右键 → 搜索 "LTXText" |
| **参数** | `text_encoder_name`: 选择 `gemma_3_12B_it_fp4_mixed.safetensors` |
| **输出** | `text_encoder` → LTXVTextToVideoSampler |

### 节点 3：EmptyLTXVLatentVideo

| 参数 | 推荐值 | 理由 |
|------|:------:|------|
| `width` | 832 | 需为 32 的倍数 |
| `height` | 480 | 需为 32 的倍数 |
| `length` | 33 | 需满足 8×N+1 |
| `batch_size` | 1 | — |

### 节点 4-5：CLIP Text Encode（正面/负面提示词）

正面提示词推荐格式：`[场景描述]，[运动描述]，[风格描述]，[画质描述]`

```
Cinematic shot of a woman walking through a rainy Tokyo street at night, 
holding a transparent umbrella, camera slowly pans left, 4K, high quality
```

负面提示词模板：
```
worst quality, low quality, blurry, distorted, ugly, bad anatomy, 
deformed, disfigured, watermarked, text, logo, cropped, jpeg artifacts
```

### 节点 6：LTXVTextToVideoSampler

| 参数 | 推荐值 | 理由 |
|------|:------:|------|
| `seed` | -1 或固定 | -1 随机，固定可复现 |
| `steps` | 30-50 | 低于 20 步运动不连贯 |
| `cfg` | 3.0-4.0 | 过高导致运动僵硬 |
| `sampler_name` | euler | LTX 原生测试采样器 |
| `scheduler` | normal | 配合 euler |

### 节点 7：VAEDecodeTiled（分块 VAE 解码）

| 项目 | 说明 |
|------|------|
| **参数** | `tile_size`: 512（默认） |
| **理由** | 分块解码减少显存峰值 |

### 节点 8：SaveVideo

| 项目 | 说明 |
|------|------|
| **参数** | `fps`: 24 或 30（推荐 24） |

- [ ] 提示词已填写
- [ ] 分辨率宽高为 32 的倍数
