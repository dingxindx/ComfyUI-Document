# LTX 2.3 文生视频工作流

> 这是最基础的视频工作流。跑通这个再学两阶段和高端玩法。

## 一、完整节点清单（8 个节点）

| # | 节点名称 | 作用 | 必选 |
|:-:|:---------|:-----|:----:|
| 1 | CheckpointLoaderSimple | 加载 LTX 主模型 | ✅ |
| 2 | LTXTextEncoderLoader | 加载 Gemma 文本编码器 | ✅ |
| 3 | EmptyLTXVLatentVideo | 初始化视频潜空间 | ✅ |
| 4 | CLIP Text Encode (Prompt) | 正面提示词 | ✅ |
| 5 | CLIP Text Encode (Negative) | 负面提示词 | ✅ |
| 6 | LTXVTextToVideoSampler | 核心采样器 | ✅ |
| 7 | VAEDecodeTiled | 分块 VAE 解码 | ✅ |
| 8 | SaveVideo | 保存视频 | ✅ |

## 二、节点参数详解

### CheckpointLoaderSimple
- `ckpt_name` → 选择 `ltx-2.3-22b-dev-fp8.safetensors`
- MODEL → 采样器；VAE → VAE Decode（LTX 的 VAE 内置在 checkpoint 中）

### LTXTextEncoderLoader
- 搜 "LTXText" 添加
- `text_encoder_name` → 选择 `gemma_3_12B_it_fp4_mixed.safetensors`
- 输出 → 连接采样器的 `text_encoder` 端口

### EmptyLTXVLatentVideo
| 参数 | 推荐值 | 说明 |
|:-----|:------:|:-----|
| `width` | 832 | 32 的倍数 |
| `height` | 480 | 32 的倍数 |
| `length` | 33 | 满足 8×N+1（33=4×8+1）|
| `batch_size` | 1 | 一次一段视频 |

### CLIP Text Encode (正面/负面)
正面提示词模板：`[场景描述]，[运动描述]，[风格描述]，[画质描述]`
```
Cinematic shot of a woman walking through a rainy Tokyo street at night,
holding a transparent umbrella, camera slowly pans left, 4K, high quality
```

负面提示词：
```
worst quality, low quality, blurry, distorted, ugly, bad anatomy,
deformed, disfigured, watermarked, text, logo, cropped, jpeg artifacts
```

### LTXVTextToVideoSampler
| 参数 | 推荐值 | 说明 |
|:-----|:------:|:-----|
| `seed` | -1 或固定 | -1=随机 |
| `steps` | 30-50 | <20 步运动不连贯 |
| `cfg` | 3.0-4.0 | >5.0 运动僵硬 |
| `sampler_name` | euler | 推荐 |
| `scheduler` | normal | 配合 euler |

### VAEDecodeTiled
- 与普通 VAE Decode 不同，分块解码减少显存峰值
- `tile_size`: 512（默认值）

### SaveVideo
- `fps`: 24（推荐，与训练数据一致）

## 三、连线一览

```
CheckpointLoaderSimple.MODEL → LTXVTextToVideoSampler.model
CheckpointLoaderSimple.VAE  → VAEDecodeTiled.vae
LTXTextEncoderLoader.text_encoder → LTXVTextToVideoSampler.text_encoder
EmptyLTXVLatentVideo.latent → LTXVTextToVideoSampler.samples
CLIP Text (Positive).CONDITIONING → LTXVTextToVideoSampler.positive
CLIP Text (Negative).CONDITIONING → LTXVTextToVideoSampler.negative
LTXVTextToVideoSampler.latent → VAEDecodeTiled.samples
VAEDecodeTiled.IMAGE → SaveVideo.images
```

## 四、检查清单

- [ ] 所有节点已连接，无红线
- [ ] 提示词已填写（必须是英文）
- [ ] 分辨率宽高为 32 的倍数（832×480）
- [ ] length 满足 8×N+1（33/41/49/57 等）
- [ ] 点击 Queue Prompt 能正常执行
- [ ] 视频生成需要时间——等待进度条完成
