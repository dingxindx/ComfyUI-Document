# 社区资源获取指南

## 一、三大模型社区对比

| 平台 | 特点 | 国内访问 | 用途 |
|:-----|:------|:---------|:-----|
| **HuggingFace** (huggingface.co) | 最全基座模型，官方权重 | 需镜像（hf-mirror.com）| 基础模型、ControlNet、IP-Adapter |
| **Civitai** (civitai.com) | 社区微调、LoRA 最丰富 | 需代理 | LoRA、风格模型、微调 Checkpoint |
| **哩布哩布** (liblibai.com) | 国内平台，高速下载 | 直接访问 | 国内首选，大部分 Civitai 模型被搬运 |
| **Modelscope (魔搭)** (modelscope.cn) | 阿里出品，中文模型丰富 | 直接访问 | 通义系列、Wan 视频、中文模型 |

## 二、从图片反推工作流参数

### 方法 1：直接拖入 ComfyUI

这是最快的方法——**把图片直接拖到 ComfyUI 画布上**，它会自动提取图片中嵌入的元数据，还原整个工作流节点。

**限制**：只有生成该图片时**使用了 ComfyUI 节点保存功能**的图片才能还原。从 Civitai 下载的预览图通常都可以。

### 方法 2：查看 Civitai 元数据

1. 打开 Civitai 目标图片 → 点击图片下方的 **"Show Metadata"**（显示元数据）
2. 元数据中包含完整的 prompt（正/负面）、steps、CFG、seed、model、LoRA 列表、分辨率
3. 记下这些参数，在 ComfyUI 中手动搭建

### 方法 3：反推+还原（乌鸦反推）

用 `ComfyUI-Impact-Pack` 的 **ReActor** 或 **Reverse Image Engine** 节点可以分析图片的"风格参数"（不含嵌入元数据的图片也能分析），但这通常只能得到大致相近的 prompt，不如元数据精确。

### 采样器名称对照（Civitai → ComfyUI）

Civitai 显示的名称和 ComfyUI 中实际的参数值存在名称差异：

| Civitai 显示 | ComfyUI sampler_name | ComfyUI scheduler |
|:-------------|:--------------------:|:-----------------:|
| DPM++ 2M Karras | dpmpp_2m | karras |
| Euler a | euler | ancestral |
| DPM++ 2M SDE Karras | dpmpp_2m_sde | karras |
| Euler | euler | normal |
| DPM++ 2S a Karras | dpmpp_2s | ancestral |
| DPM++ 3M SDE Karras | dpmpp_3m_sde | karras |
| DPM++ SDE Karras | dpmpp_sde | karras |
| LMS | lms | normal |
| UniPC | uni_pc | normal |

## 三、模型搜索技巧

1. 去 Civitai/哩布哩布 搜你想做效果的关键词
2. 看该图的 **Recipe** 区域（Civitai 上通常在图片右侧），会列出 Base Model + LoRA + CFG + Steps + Resolution
3. **复制那套组合**，不要自己猜参数——这是最省事的路径
4. 如果有些参数没列（如 scheduler），用默认值（euler + normal）

## 四、国内下载策略速查

| 资源 | 方案 |
|:-----|:------|
| HuggingFace 模型 | `export HF_ENDPOINT=https://hf-mirror.com` |
| GitHub 节点 | `git clone https://gitclone.com/github.com/...` |
| Civitai LoRA | 去 哩布哩布 搜同名 |
| pip 依赖 | `pip install -i https://pypi.tuna.tsinghua.edu.cn/simple` |
| Modelscope 模型 | 直接下载，阿里源国内很快 |
| 百度网盘分享 | 经常有整合包分享，但更新不及时 |

## 五、如何分辨模型类型

下载模型前，确认它是你需要的类型：

| 文件大小 | 大概率是 | 存放位置 |
|:--------:|:---------|:---------|
| 2-7GB | Checkpoint（主模型） | `models/checkpoints/` |
| 30-150MB | LoRA | `models/loras/` |
| 10-50KB | Textual Inversion 嵌入 | `models/embeddings/` |
| 1-2GB | ControlNet / VAE | `models/controlnet/` / `models/vae/` |
| 100MB | IP-Adapter | `models/ipadapter/` |
