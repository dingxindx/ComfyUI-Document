# 文生图工作流——从零到精通的完整教学

> **前置条件**：已完成 `00-小白上路/00-03-下载第一个模型并首次生成.md` 的内容，即你已经成功生成过一张图片。

本章的任务是用一个完整的例子，让你彻底搞懂文生图工作流的每一个节点、每一个参数、每一种可能的错误。

---

## 一、你要搭建的工作流长什么样？

```
CheckpointLoaderSimple ───┬── MODEL ──→ KSampler
                          ├── CLIP ────→ CLIP Text Encode (Prompt)
                          │           └→ CLIP Text Encode (Negative)
                          └── VAE ─────→ VAE Decode
                                          ↑
Empty Latent Image ─────── LATENT ───→ KSampler
                                          ↓
                                        VAE Decode → Save Image
```

总共 **6 个节点**（CheckpointLoaderSimple, CLIP Text Encode x2, Empty Latent Image, KSampler, VAE Decode, Save Image）。

---

## 二、逐个节点详解

### 节点 1：CheckpointLoaderSimple（模型加载器）

**添加方式**：在画布空白处右键 → 输入 `checkpoint` → 选择 `CheckpointLoaderSimple`

**这个节点在做什么**：从硬盘读取 checkpoint 文件，加载三个组件（UNET、CLIP、VAE）到显存中。

**参数设置**：

```
┌────────────────────────────────┐
│ CheckpointLoaderSimple         │
│  ckpt_name:  dreamshaper_8     │ ← 下拉选择你下载的模型
└────────────────────────────────┘
```

| 参数 | 说明 | 可选值 |
|:-----|:-----|:-------|
| `ckpt_name` | 选择模型文件 | 下拉列表显示 `models/checkpoints/` 目录中的所有模型 |

**⚠️ 常见问题**：
- **下拉框是空的** → 模型没放到 `models/checkpoints/` 目录
- **点了没反应** → 刷新浏览器（F5）或重启 ComfyUI
- **节点变红** → 模型文件损坏或不完整，重新下载

---

### 节点 2-3：CLIP Text Encode（文本编码器——正面和负面提示词）

**添加方式**：右键 → 输入 `CLIP Text Encode` → 选择 `CLIP Text Encode (Prompt)`

**你需要两个这样的节点**——一个写正面提示词（positive），一个写负面提示词（negative）。

**简便操作**：
1. 添加第一个 CLIP Text Encode
2. 选中它，按 Ctrl+C 复制，Ctrl+V 粘贴
3. 把粘贴出来的副本的参数改一改（改提示词和重新连线）

**这个节点在做什么**：把你的文字描述翻译成 CLIP 模型能理解的数学向量，然后在潜空间里画圈圈。

**参数设置**：

```
┌────────────────────────────────┐
│ CLIP Text Encode (Prompt)      │
│  text:                         │
│  ┌──────────────────────────┐  │
│  │ a cute cat sitting on    │  │
│  │ a table, wearing a       │  │
│  │ wizard hat, detailed     │  │
│  └──────────────────────────┘  │
└────────────────────────────────┘
```

**正面提示词 (Positive Prompt)**：描述你要生成的内容。越具体越好。

**写法模板**：`[主体描述]，[环境/背景]，[风格]，[画质关键词]`

```
a cute cat sitting on a wooden table, wearing a wizard hat with stars,
soft studio lighting, detailed fur, 8K, high quality, photorealistic
```

**负面提示词 (Negative Prompt)**：描述你不要的内容。通用的负面提示词可以复制保存下来反复用。

```
ugly, blurry, low quality, distorted, deformed, bad anatomy, 
extra fingers, extra limbs, fused fingers, missing fingers, 
poorly drawn hands, poorly drawn face, mutation, disfigured, 
out of frame, text, watermark, logo, signature
```

> 💡 **什么是 CLIP？** CLIP 是 OpenAI 训练的"图片-文字"匹配模型。它学会了「猫」这个字的向量和猫图片的向量在数学空间里很接近。所以你的 prompt 经过 CLIP 编码后，UNET 就知道该往"猫"的方向去噪。

**⚠️ 常见问题**：
- **写中文 prompt 效果不好** → 大部分模型用英文训练，中文理解差。建议用英文
- **节点变红** → 检查 `clip` 端口是否从 CheckpointLoaderSimple 的 CLIP 端口连接了
- **生成的图完全不相关** → prompt 可能太模糊，加具体描述词

---

### 节点 4：Empty Latent Image（空潜空间初始化）

**添加方式**：右键 → 输入 `Empty Latent Image` → 选择

**这个节点在做什么**：创建一个装满噪声的"空白画布"。这个噪声是纯随机的，UNET 会在这个噪声上一步步去噪，最终形成图片。

**你可以理解为**：一张白纸上撒满了彩色灰尘（噪声），AI 的工作是"从灰尘中还原出图片"。

**参数设置**：

```
┌────────────────────────────────┐
│ Empty Latent Image             │
│  width:       512              │
│  height:      512              │
│  batch_size:  1                │
└────────────────────────────────┘
```

| 参数 | 推荐值 | 说明 |
|:-----|:------:|:-----|
| `width` | 512 (SD1.5) / 1024 (SDXL) | 图片宽度，必须是 64 的倍数 |
| `height` | 512 (SD1.5) / 1024 (SDXL) | 图片高度，必须是 64 的倍数 |
| `batch_size` | 1 | 一次生成几张，1 是单张 |

**为什么宽高要 64 的倍数？**
因为潜空间的大小是像素大小 / 8（VAE 编码器的下采样率是 8）。如果宽高不是 8 的倍数，经过 VAE 编码后会有尺寸不匹配的问题。ComfyUI 要求 64 的倍数更保险。

**⚠️ 常见问题**：
- **图片太花了/变形了** → 分辨率和模型不匹配。SD1.5 用 512×512，用 768×768 效果已经变差
- **生成了黑色图片** → VAE 连接问题（但不是本节点的问题）

---

### 节点 5：KSampler（采样器——核心节点）

**添加方式**：右键 → 输入 `KSampler` → 选择

**这个节点在做什么**：这是整条工作流的"心脏"。它接受：
- 模型（UNET）
- 条件（prompt 编码）
- 初始噪声（潜在空间）

然后运行"去噪"过程——从噪声开始，每一步根据 UNET 的预测小幅度去噪，最终得到清晰的潜空间图像。

```
初始噪声 → 第 1 步 → 第 2 步 → ... → 第 20 步 → 最终潜空间
(纯雪花)    (隐约轮廓)              (清晰图像)
```

**参数设置**：

```
┌────────────────────────────────┐
│ KSampler                       │
│  seed:           -1            │
│  steps:          20            │
│  cfg:            7.0           │
│  sampler_name:  euler         │
│  scheduler:     normal        │
│  denoise:       1.0           │
└────────────────────────────────┘
```

**逐参数详解**：

| 参数 | 推荐值 | 范围 | 详细说明 |
|:-----|:------:|:----|:---------|
| `seed` | -1 或固定数字 | -1~亿 | 随机数种子。**-1=每次随机**，相同数字=复现相同结果。如果你生成了一个非常满意的图，记下 seed 值，下次固定它就能复现 |
| `steps` | 20 | 1-150 | 去噪步数。**越多越精细，但回报递减。** SD1.5 的甜点是 20-30 步，超过 50 步效果提升几乎不可见。可理解为一幅画从涂鸦到精修的画笔次数 |
| `cfg` | 7.0 | 1-30 | **提示词相关度（Classifier-Free Guidance）**。值越高，AI 越"听"你的 prompt。7 是标准值。**太高（15+）= 过饱和、颜色溢出。** **太低（1-3）= 自由创作，但可能不相关** |
| `sampler_name` | euler | 多种 | 采样算法。**euler** 最经典、稳定。**dpmpp_2m** 更快更好但偶尔不稳定。初学者用 euler 不会出错 |
| `scheduler` | normal | 多种 | 调度器，配合采样算法使用。euler + normal 是最小化学习曲线组合 |
| `denoise` | 1.0 | 0-1 | **降噪强度**。**文生图固定 1.0**（从纯噪声开始）。**图生图时降低**（0.3-0.8）可控制对原图改变程度 |

**⚠️ 常见问题**：
- **图片颜色过度饱和** → cfg 太高，降低到 7 以下
- **图片模糊/缺乏细节** → steps 太少（<10）或 cfg 太低（<4）
- **图片和 prompt 无关** → cfg 太低或者 prompt 太短
- **生成失败/报错** → 检查所有端口是否连接正确

---

### 节点 6：VAE Decode（VAE 解码器）

**添加方式**：右键 → 输入 `VAE Decode` → 选择

**这个节点在做什么**：把潜空间（64×64 的数字矩阵）**解压**回像素空间（512×512 的 RGB 图片）。没有它，你永远看不到最终的图片。

```
潜空间 (64×64) ───→ VAE 解码 ───→ 像素图 (512×512)
  蓝色端口                   绿色端口
```

**参数设置**：不需要设置任何参数。只需连接两个输入端口：
- `samples` ← KSampler 的 `LATENT`（蓝色接蓝色）
- `vae` ← CheckpointLoaderSimple 的 `VAE`（黄色接黄色）

**⚠️ 常见问题**：
- **图片全黑/全绿/全灰** → VAE 端口没有正确连接。检查 VAE 黄色线是否从 Checkpoint 连接
- **图片有格子状伪影** → 说明模型本身产生了问题，通常不是因为 VAE Decode

---

### 节点 7：Save Image（保存图片）

**添加方式**：右键 → 输入 `Save Image` → 选择

**这个节点在做什么**：把 VAE 解码出来的像素图片保存到硬盘上的 `output/` 目录。

**参数设置**：

| 参数 | 建议值 | 说明 |
|:-----|:------:|:-----|
| `filename_prefix` | "ComfyUI" | 输出文件的文件名前缀，生成的文件名为 `前缀_序号_时间戳.png` |

**保存位置**：`ComfyUI/output/` 目录。在文件资源管理器中打开这个目录，就能看到生成的图片。

**预览**：ComfyUI 界面中也会显示生成结果，不需要专门打开图片文件来看。

## 三、完整连线步骤（把一切拼起来）

在完成所有节点添加后，按以下步骤连线：

### 第 1 步：连接模型

从 **CheckpointLoaderSimple** 拖出 3 根线：
1. `MODEL`（紫色）→ KSampler 的 `model`（紫色）
2. `CLIP`（粉色）→ **两**个 CLIP Text Encode 节点的 `clip`（粉色）（一个输出接两个输入）
3. `VAE`（黄色）→ VAE Decode 的 `vae`（黄色）

### 第 2 步：连接提示词

从 **CLIP Text Encode (Prompt)**（正面）→ 拖出 `CONDITIONING`（橙色）→ KSampler 的 `positive`（橙色）

从 **CLIP Text Encode (Negative)**（负面）→ 拖出 `CONDITIONING`（橙色）→ KSampler 的 `negative`（橙色）

### 第 3 步：连接潜空间

从 **Empty Latent Image** → 拖出 `LATENT`（蓝色）→ KSampler 的 `latent_image`（蓝色）

### 第 4 步：连接采样→解码

从 **KSampler** → 拖出 `LATENT`（蓝色）→ VAE Decode 的 `samples`（蓝色）

### 第 5 步：连接输出

从 **VAE Decode** → 拖出 `IMAGE`（绿色）→ Save Image 的 `images`（绿色）

### 连线检查清单

- [ ] 每个端口都**有且只有一根**线连接
- [ ] 所有线都是**白色**的（红色=端口类型不匹配）
- [ ] CheckpointLoaderSimple 的 MODEL/CLIP/VAE 三个端口都连接出去了
- [ ] KSampler 的所有输入端口（model, positive, negative, latent_image）都连上了
- [ ] 没有多余的未连接端口

## 四、调试：如果生成了不对劲的图片

| 现象 | 原因 | 怎么修 |
|:-----|:-----|:-------|
| 全黑图片 | VAE 端口没连接 | 检查 VAE 黄色线 |
| 全彩色噪点 | 模型加载失败 | 重启 ComfyUI |
| 绿色/紫色图片 | VAE 不兼容 | 模型自带的 VAE 应该有兼容性 |
| 过于饱和 | cfg 太高 | 降到 5-7 |
| 模糊/像雾 | steps 太少或 cfg 太低 | steps 加到 25-30，cfg 加到 7-8 |
| 多出诡异的人体部位 | 负面提示词不够 | 加负面提示词（extra fingers, fused limbs 等）|
| 和描述完全不符 | prompt 太短或 cfg 太低 | 加长描述，cfg 提高到 7 |
| 人像脸部扭曲 | 模型问题或步骤不够 | 增加 steps 到 30+，检查模型 |

---

## 五、总结——你现在已经掌握的知识

做完这个工作流后，你应该理解：

1. **Checkpoint 是什么** → 一个 .safetensors 文件里包含 UNET/CLIP/VAE
2. **Latent Space 是什么** → AI 在缩小的潜空间里工作，比像素空间快 48 倍
3. **VAE 的作用** → 连接潜空间和像素空间的唯一桥梁
4. **KSampler 的参数含义** → steps（画笔次数）、cfg（听话程度）、seed（随机种子）
5. **CLIP Text Encode** → 把文字翻译成数学向量
6. **为什么同色端口才能连接** → 每种颜色代表不同的数据类型
